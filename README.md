# ЖИЖА — генератор живых процедурных фонов

Веб-приложение на чистом фронтенде: генератор анимированных «жидких» фонов на WebGL.
10 типов процедурного поля, surface-эффекты (пиксель / гекс / ASCII / полутон), доменное
искажение (domain warp), интерактивная рябь от курсора, шеринг сцены по ссылке и embed-код,
экспорт PNG и запись видео — всё в одном `index.html`, без сборки и зависимостей-пакетов.

**Демо:** включите GitHub Pages для этого репозитория (ветка `main`, корень `/`) — приложение откроется на `https://<user>.github.io/<repo>/`.

> **v2.** Движок переписан: value-noise GLSL (precision-safe, стабилен на мобильных и в долгих
> сессиях) в чистой Three.js-оболочке с render-on-demand. Предыдущая версия сохранена в
> `index.v1.backup.html`.

## Возможности

- **WebGL / Three.js** (через CDN) + кастомный GLSL fragment-шейдер.
- **10 типов поля:** Поток, Течение (curl-flow), Клетки (Worley), Гироид, Труше, Интерференция, Калейдоскоп, Линии, Сетка, Золото (phyllotaxis).
- **Поверхность (surface):** Квадрат / Гекс-сетка / ASCII-атлас + независимая пикселизация и полутон (halftone) — пикселизация и точки **складываются** (как в FLUID: Square + pixelate + halftone).
- **16 шаблонов:** Бореалис, Послесвет, Прилив, Магма, Туманность, Плазма, Бархат, Оникс, Плетение, Контур, Рябь, Мандала, Страты, Решётка, Подсолнух, Гидрополе.
- **Параметры поля:** скорость, масштаб (zoom), искажение (warp), зернистость (grain).
- **Цвета:** 8 палитр (Сияние, Закат, Океан, Сумерки, Уголь, Мята, Ирис, процедурный Хром) + кастомный 4-стоповый градиент.
- **Размер:** пресеты 9:16, 4:5, 1:1, 5:4, 3:2, 16:9 + бесшовный (tileable) режим с настройкой числа плиток.
- **Интерактив:** движение курсора создаёт локальную рябь/завихрение потока.
- **Режимы:** «Анимация» (Live) и «Статика» (Freeze).
- **Шеринг:** ссылка кодирует всю сцену в URL (`#s=...`); кнопка «Код для встраивания» даёт готовый `<iframe>` (режим `embed=1` скрывает панель).
- **Импорт ссылок FLUID:** вставьте ссылку `fluid.krackeddevs.com/#p=...` (формат «Share for Claude») — сцена воспроизводится 1-в-1 (поле, палитра, surface, pixelate, halftone, seed, формат).
- **Фон на чужом сайте:** готовый набор в [`embed/`](embed/EMBED.md) — универсальный веб-компонент `<fluid-bg>` (HTML / React / Next / Vue / Svelte) + статичный фоллбэк. В embed-режиме холст растягивается на весь экран (full-bleed, без полос).
- **Экспорт:** PNG (Эскиз / 1080p / 2048² / 4K) и запись видео WebM прямо с холста.

## Производительность

Приложение бережёт ресурсы:

- **Render-on-demand** — в режиме «Статика» без взаимодействия кадры не рисуются (≈0 нагрузки на GPU).
- **Кадры ограничены 60 FPS** — нет смысла гнать медленную жидкость быстрее даже на 120-Гц экранах.
- **Пауза при скрытой вкладке** (`visibilitychange`) — никакого фонового расхода.
- **Pixel ratio ≤ 1.5** — на retina-экранах режет число пикселей без видимой потери качества.
- **Precision-safe hash + bounded drift** — координаты шума «орбитят», а не растут бесконечно, поэтому даже в многочасовой сессии float-точность не вырождается в мерцание (главная причина, почему долгие анимации не «глючат»).
- Шейдер оптимизирован: фиксированные 5 октав fbm, подсветка глубины переиспользует уже посчитанное поле.

## Запуск локально

Любой статический сервер из корня репозитория, например:

```bash
python3 -m http.server 4173
# открыть http://localhost:4173/
```

Или просто открыть `index.html` в браузере (file://). Для записи видео WebM нужен Chrome или Edge.

## Стек

HTML5 · CSS3 (glassmorphism) · Vanilla JS (ES modules) · Three.js · GLSL (value-noise fbm).
Внешних npm-зависимостей нет — Three.js подключается с CDN через import map.
