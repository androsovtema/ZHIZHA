# Prompt For Claude Design: UX-редизайн ЖИЖА

Ты — senior product designer / UX architect. Твоя задача — сделать полный UX-редизайн сервиса ЖИЖА.

Важно: это НЕ косметический редизайн. Не начинай с цветов, шрифтов, эффектов, красивых карточек и визуального стиля. Главная задача — переосмыслить логику продукта, пользовательские сценарии, информационную архитектуру, порядок действий, структуру экранов, приоритеты функций и удобство достижения цели.

## Контекст продукта

ЖИЖА — это генератор живых процедурных фонов для видео, сторис, обложек, презентаций, сайтов и абстрактных лупов.

Пользователь приходит не "покрутить WebGL-настройки", а решить задачу:

- быстро сделать красивый фон для TikTok/Reels/Stories;
- скачать видеофон;
- скачать PNG для обложки/поста;
- сделать спокойный фон под текст;
- сделать бесшовный паттерн;
- получить фон для сайта;
- быстро подобрать шаблон и немного настроить результат.

Сейчас в проекте уже есть рабочая логика, Simple/Pro режимы, шаблоны, категории, экспорт, embed, URL sharing и WebGL-preview. Но UX нужно перестроить так, чтобы пользователь максимально быстро понимал, что делать, видел результат и доходил до экспорта без лишней когнитивной нагрузки.

Перед проектированием внимательно изучи файл ZHIZHA_UX_INVENTORY.md и сам файл проекта index.html

Также используй прикрепленные скриншоты.

Этот документ — источник правды о текущем устройстве продукта: какие есть режимы, настройки, сценарии, проблемы, ограничения и возможности.

## Главная задача

Спроектируй новый UX ЖИЖА так, чтобы сервис стал creator-first инструментом:

```text
цель пользователя -> подходящие шаблоны -> быстрая настройка -> понятный экспорт
```

Не надо удалять мощные возможности. Нужно правильно расставить уровни:

1. Быстрый пользовательский слой — для creator-а.
2. Расширенный слой — для точной настройки.
3. Developer/site слой — для embed, iframe, ссылки, web component.
4. Системный слой — тема UI, служебные настройки, performance/debug.

## Что нужно улучшить в UX

Обязательно переосмысли:

- стартовый экран;
- порядок выбора: формат / цель / шаблон / настройка / экспорт;
- мобильный сценарий;
- длинную ленту шаблонов;
- место быстрых настроек;
- место экспортных действий;
- поведение категорий;
- разницу между Simple и Pro;
- названия каждой из настроек;
- то, как пользователь понимает "что изменится" при выборе категории;
- паттерны и бесшовный режим;
- экспорт видео vs PNG;
- фон под текст и проверку читаемости;
- сайт/embed-функции;
- путаницу между темой интерфейса и палитрой сцены;
- способ объяснить Pro-настройки без перегруза.

## Важные UX-проблемы из текущего продукта

Учитывай эти проблемы как обязательные:

- На mobile пользователь долго скроллит через шаблоны, прежде чем добраться до быстрых настроек и экспорта.
- Категория TikTok/Reels говорит, что видео важно, но первой кнопкой экспорта остается PNG.
- Категория Паттерн обещает бесшовность, но seamless не включается автоматически в Simple.
- Для Бесшовный паттерн доступны не все варианты из Поле. Точнее для клика доступны все, но переключаясь по большинсту ничего в превью фона не меняется. Надо либо сделать кнопки неактивными для тех, кто не поддатется бесшовности, либо в целом их скрыть при включенном Бесшовный паттерн.
- Export не адаптируется к выбранной цели.
- Simple и Pro используют разные термины для одних параметров.
- Developer/embed функции спрятаны и не показывают лучший способ встраивания `<zhizha-bg>`.
- Тема интерфейса и палитра фона визуально могут восприниматься как одна настройка.
- Сервис обещает "живые фоны за 20 секунд", но текущий flow иногда длиннее.

## Что нужно сохранить

Не ломай продуктовую суть. Сохрани в новой UX-логике:

- live preview как главный центр интерфейса;
- категории по назначению;
- шаблоны;
- quick controls;
- Pro-настройки;
- PNG export;
- video export;
- scene URL sharing;
- import scene from URL;
- embed mode;
- iframe export;
- `<zhizha-bg>` web component;
- mobile sticky preview;
- возможность точной настройки поля, фактуры, цвета, размера и бесшовности.

Сделай не просто красивый макет, а полноценную UX-концепцию.

Нужен результат в таком формате:

## Особое внимание

Не делай лендинг. Первый экран должен быть рабочим инструментом.

Не делай редизайн ради красоты. Каждый элемент должен отвечать на вопрос:

- помогает ли он пользователю быстрее получить фон?
- уменьшает ли он неопределенность?
- приближает ли он к экспорту?
- объясняет ли он результат действия?
- не мешает ли live preview?

## Целевая UX-модель

Идеальный путь пользователя должен быть примерно таким:

```text
Открыл ЖИЖА
-> выбрал задачу: Reels / Stories / YouTube / Фон под текст / Обложка / Паттерн / Сайт (или стоит рассмотреть изменение/добавление задач?)
-> сразу получил подходящий формат и recommended export
-> выбрал шаблон
-> быстро подкрутил цвет, движение, детализацию
-> скачал видео/PNG или скопировал embed
```

## Принцип проектирования

ЖИЖА должна ощущаться не как "панель параметров шейдера", а как быстрый creator-инструмент уровня CapCut/Lightroom/Canva, но с живым процедурным фоном. В будущем функционал будет расширяться, будет все больше и больше вариантов шаблонов, фона, настроек, инструментов, поэтому проект должен быть масштабируемым.

Pro-возможности должны остаться, но не должны быть первым, чтобы пользователь был вынужден разбиратсья.
