# Handoff: Редизайн ЖИЖА (UX + система тем)

> Передача дизайн-концепции для реализации в Claude Code. Этот README **самодостаточен**: разработчик, не участвовавший в проектировании, может реализовать всё по нему одному. Дизайн-файлы в комплекте — справочные.

---

## Overview

**ЖИЖА** — веб-платформа генерации живых (анимированных) фонов для контент-мейкеров: Reels/Stories/обложки, фоны под текст, бесшовные паттерны и встраиваемые фоны для сайтов. Движок (WebGL-шейдеры, рекордер видео, PNG-экспорт, seamless-тайлинг) уже работает — **переделывается оболочка/UX, не движок**.

Редизайн преследует две связанные цели:
1. **Новый UX** — превратить «панель шейдера» в creator-инструмент: задача → шаблон → подстройка → экспорт, с живым превью в центре и всегда доступным экспортом.
2. **Новая система тем оформления** — уйти от единственной «сладкой» розово-фиолетово-мятной палитры к системе из 4 тем UI. Основная и дефолтная — **Вольт**.

Шрифты и tone of voice **не меняются** (Comfortaa + Nunito, обращение на «ты», глагол вперёд, коротко).

## About the Design Files

Файлы в этом бандле — **дизайн-референсы, сделанные в HTML**. Это прототипы, показывающие задуманный вид и поведение, **а не продакшн-код для копирования**.

Задача — **воссоздать эти дизайны в среде вашего кодовой базы** (React/Vue/Svelte/web-components — что уже используется в ЖИЖА), по её устоявшимся паттернам и библиотекам. Если фронт-среды ещё нет — выбрать наиболее подходящий фреймворк и реализовать там. HTML из бандла **не отгружать в прод как есть**: переносим смысл, структуру, токены и поведение.

Дизайн сделан как набор статических документов с макетами. Интерактивность в реальном приложении (запись видео, экспорт, переключение тем, drag-слайдеры) реализуется средствами кодовой базы — в макетах она показана визуально.

## Fidelity

**High-fidelity (hifi).** Цвета, типографика, радиусы, тени, отступы — финальные и точные (см. Design Tokens). Воссоздавайте UI попиксельно средствами кодовой базы. Где в макете показано схематичное превью фона (CSS-градиенты) — это **плейсхолдер живого WebGL-холста**, его рисует существующий движок, не копируйте градиенты буквально.

## Система тем (ядро редизайна)

4 темы. **Меняется только цветовой слой** — шрифты, голос, радиусы, отступы, тени, движение, структура компонентов общие для всех тем.

| Тема | id | Роль | Сигнатурные цвета |
|---|---|---|---|
| **Вольт** | `volt` | **Дефолт + приоритет.** Лицо бренда, маркетинг, скриншоты, OG. | лайм `#C6FF36` → мята `#00E6C0` → циан `#00B3FF` |
| **Карамель** | `karamel` | Наследие (старая палитра, сохранена 1:1) | `#FF5FA8` · `#8A5CFF` · `#2FE6C8` |
| **Магма** | `magma` | Тёплый огонь | `#FFB627` · `#FF5E3A` · `#E8156F` |
| **Хром** | `chrome` | Холодный графит + электрик-синий | `#8FB4FF` · `#3B82FF` · `#1E5BFF` |

### Реализация тем
- Один набор имён CSS-переменных, разные значения по теме. Тема выбирается атрибутом `data-theme` на корне (`<html>` или корневом контейнере приложения).
- **Дефолт** при отсутствии сохранённого значения — `volt`.
- Выбор пользователя **персистентный**: `localStorage['zhz-theme']`, читается на старте.
- Переключение — смена одного атрибута, без перезагрузки; мягкий кросс-фейд `0.45s ease` на `background-color/color/border-color/box-shadow`.
- **Тема UI ≠ палитра сцены.** Тема UI — оформление интерфейса (живёт в системном слое, в углу шапки, с подписью «Тема UI»). Палитра сцены — цвета самого фона (живёт в настройках фона, слой «Быстро»/«Профи»). В текущем продукте их путают — разводим.

## Продуктовая логика — 4 слоя сложности

Прогрессивное раскрытие. 90% пользователей живут в слое 1.

1. **Быстрый слой (видим всегда):** Задача (контекст-чип) · карусель шаблонов + фильтр · быстрая настройка (5 контролов) · action-док экспорта.
2. **Расширенный (по запросу):** формат · бесшовность + плитки · тип поля · скорость/масштаб/движение/зерно · фактура · палитра + кастомные стопы.
3. **Для сайта (по запросу):** web-компонент `<zhizha-bg>` (рекомендованный) · iframe · ссылка на сцену · импорт сцены · статичный fallback.
4. **Системный (в углу):** тема UI · ширина панели · производительность/FPS · reduced motion · debug.

**Принцип-фильтр для любого решения:** быстрее к фону? · меньше неопределённости? · ближе к экспорту? · не мешает превью?

---

## Основной flow

`Задача → Шаблон → Подстройка → Экспорт`, **превью живёт сквозь все шаги** (постоянный центр, не «следующий экран»). Шаги не блокируют: экспорт-док закреплён, поэтому экспорт доступен с любого шага. После выбора задачи сразу есть живое превью дефолтной сцены.

## Задачи (категории) — каждая это пресет

Задача за один тык настраивает: **формат + фильтр шаблонов + рекомендованное действие экспорта + поведенческие умолчания**. Добавление новой задачи = одна строка пресета, без перестройки UI.

| Задача | Формат | Рекоменд. экспорт | Поведение по умолчанию |
|---|---|---|---|
| Reels / TikTok | 9:16 | **Видео MP4** | движение бодрее, видео-кнопка дока первой |
| Сторис | 9:16 | **Видео MP4** | спокойнее, центр свободен под текст |
| YouTube | 16:9 | **PNG обложка** | текст слева, правый-низ свободен |
| Фон под текст | 16:9 | **PNG / Видео** | оверлей читаемости ВКЛ., движение ↓ |
| Обложка | 4:5 | **PNG** | кадр «заморожен», зерно умеренное |
| Паттерн | 1:1 | **PNG-плитка** | бесшовность ВКЛ. сразу, нетайлящиеся поля скрыты |
| Сайт | авто | **`<zhizha-bg>`** | открывает слой «Для сайта», код наготове |

> Критично: **Паттерн авто-включает seamless** (сейчас обещает, но не включает). **Фон под текст** включает оверлей читаемости.

## Единый язык параметров (Simple ↔ Pro мост)

Simple и Pro синхронят один state, но называли параметры по-разному. Делаем человеческое имя каноничным, техническое — подписью в Pro.

| Было · Simple | Было · Pro | Стало · единое (канон + тех-подпись) |
|---|---|---|
| Движение | Искажение | **Движение** (warp) |
| Детализация | Масштаб | **Детализация** (scale) |
| Эффект → Нет | Фактура → Квадрат | **Фактура → Гладкая** |
| Скорость | Скорость | **Скорость** (speed) |

**5 контролов быстрого слоя:** Цвет (палитра) · Скорость · Движение · Детализация · Фактура. **Мост:** при переходе в Профи задача и шаблон остаются видны вверху, ползунки — с теми же значениями, ничего не сбрасывается.

## Экспорт — адаптивный док

Главное действие = действие активной задачи; остальное рядом вторичными кнопками.
- **Reels/Сторис:** «● Записать видео» крупно → PNG, Ссылка рядом.
- **Обложка/YouTube:** «↓ Скачать PNG» крупно → Видео, Ссылка рядом.
- **Сайт:** «</> Скопировать код» крупно → iframe, PNG рядом.

Техническая мелочь (MP4/WebM, GPU) — в свёрнутом «?», не на свету. Разрешение/длительность — компактный чип в доке, тык → поповер. Показывать реальные пиксели перед скачиванием.

## Слой «Для сайта» — `<zhizha-bg>` первым

Сейчас UI даёт только iframe, хотя рекомендованный способ — web-компонент. Порядок: **web-компонент (основной) → iframe → ссылка на сцену**.

```html
<!-- подключи один раз -->
<script src="https://zhizha.app/zhizha-bg.js"></script>

<!-- фон-слой за контентом -->
<zhizha-bg src="https://zhizha.app/#s=…&embed=1"
           fallback="/bg.jpg" z="-1"></zhizha-bg>
```
Встроены по умолчанию: `pointer-events:none`, `loading:lazy`, `aria-hidden`. Кнопка «Скопировать» даёт тост подтверждения.

## Interactions & Behavior

- **Выбор задачи:** обновляет формат холста, фильтр шаблонов, приоритет/набор кнопок дока, поведенческие умолчания. Мгновенно.
- **Выбор шаблона:** превью переходит к сцене с пружиной (`--ease-goo`, `--dur-3`); активная карточка — бордер `--accent` + `--glow`.
- **Слайдеры подстройки:** live-обновление холста на drag (без «применить»).
- **Shuffle (↻):** случайные значения в рамках текущего шаблона/задачи.
- **Запись видео:** кнопка → состояние записи (красная точка + таймер) → «Стоп» → exporting (прогресс) → «Готово. Лови файл».
- **Копирование кода/ссылки:** тост «В буфере» + смена иконки кнопки на 2с.
- **Переключение темы UI:** `data-theme` на корне + запись в localStorage; кросс-фейд `0.45s ease`.
- **Bottom-sheet (mobile):** открытие по «Все», свайп-вниз/грабер — закрытие.
- **Responsive:** desktop — двухколоночный grid (панель 340px + превью); mobile — одноколоночный стек с закреплёнными сверху превью+док. Брейкпоинт ориентировочно ~860px.

## State Management

Переменные состояния (имена ориентировочные — подстройте под кодовую базу):
- `goal` — активная задача (enum из таблицы задач). Дефолт — последний использованный или `reels`.
- `templateId` — выбранный шаблон.
- `filter` — активный фильтр-чип шаблонов.
- `layer` — активная вкладка-слой: `templates | quick | pro | site`.
- `scene` — параметры сцены: `{ palette, speed, motion(warp), detail(scale), texture, field, grain, seamless, tiles, stops, ... }`. **Единый объект**, читаемый и Simple, и Pro.
- `uiTheme` — `volt | karamel | magma | chrome`; персист в `localStorage['zhz-theme']`, дефолт `volt`.
- `exportState` — `idle | recording | exporting | done | error | unsupported`; для записи — `elapsed` (таймер).
- `readabilityOverlay` — bool, авто-ВКЛ для задачи «Под текст».

Триггеры переходов — см. Interactions. Производные значения (формат, набор кнопок дока, видимость полей) **вычисляются из `goal`**, не дублируются в state.

## Состояния UI (8 — все нужны)

1. **Пусто** — «Выбери задачу — и поехали», сразу живое превью дефолта (плитка `--glow-grad`, флоат-анимация 3s).
2. **Активно** — шаблон/чип подсвечен, превью пружинит.
3. **Запись** — pill с пульс-точкой `--danger` + таймер, кнопка → «Стоп».
4. **Экспорт** — прогресс-бар (трек `--surface-2`, заливка `--grad`), блокировка повторного клика.
5. **Скопировано** — pill `--hi` + «✓ В буфере», 2с.
6. **Готово** — pill `--grad` + «✓ Лови файл».
7. **Ошибка** — pill `rgba(255,92,110,.12)` + бордер `--danger`: «Видео не записалось. Chrome или Edge справятся лучше».
8. **Не поддержано** — WebGL off: фоллбэк-картинка + подсказка, без белого экрана.

## Accessibility

- Все «кнопки-divы» → нативные `<button>`. Тема UI — `radio-group`, не fake-menu.
- Карусель шаблонов — стрелки + roving `tabindex`; короткая дистанция до экспорта.
- Палитры с **видимыми текст-подписями**, не только `title`.
- Контраст всех тем: текст ≥ 4.5:1, UI-элементы ≥ 3:1.
- `prefers-reduced-motion` → превью стартует статичным, есть явная кнопка «Оживить»; запись/экспорт работают и в статике.
- Состояния запись/экспорт/готово/ошибка — через `aria-live`. Таймер озвучивается дискретно, не каждую секунду.

## Design Tokens

**Все цвета — в блоках тем выше (раздел «Система тем»).** Ниже общие (тема-независимые) значения.

**Радиусы:** `--r-xs 8` · `--r-sm 12` · `--r-md 16` · `--r-lg 22` · `--r-xl 28` · `--r-pill 999`. Правило: всё интерактивное ≥ 12px; крупнее элемент — больше скругление.

**Тени / свечение:** `--shadow-card 0 8px 24px rgba(0,0,0,.35)` · `--shadow-panel 0 20px 60px rgba(0,0,0,.55)` · `--glow` (цвет за темой) — фирменный приём: action будто светится изнутри. Glow — только для действия/активности.

**Движение:** `--ease-goo cubic-bezier(.34,1.56,.64,1)` (пружина: ховеры, появление фона), `--ease-soft cubic-bezier(.4,0,.2,1)` (появления, кросс-фейд). Длительности: 120 / 200 / 320 / 600ms. Ховер пружинит, нажатие — `scale(.97)`.

**Типографика (НЕ меняется):**
| Роль | Шрифт | Размер/интерлиньяж | Где |
|---|---|---|---|
| Display | Comfortaa 700 | 40+/1.05 | hero, числа-герои |
| H1 | Comfortaa 700 | 22/1.15 | экраны, лого |
| H2 | Comfortaa 700 | 16/1.2 | названия шаблонов |
| Label | Nunito 700 | 11, letter-spacing +1.2px, uppercase | заголовки секций |
| Body | Nunito 400 | 13/1.5 | основной текст |
| UI bold | Nunito 600/700 | 11–14 | кнопки, чипы, значения |
| Code | JetBrains Mono | 11–13 | хексы, код, формат-чипы |

Comfortaa — крупные акценты и лого (используйте от 18px). Nunito — весь интерфейс. JetBrains Mono — код и технические значения.

**Tone of voice (НЕ меняется):** на «ты», глагол вперёд, коротко, дружелюбно-дерзко. Примеры из макетов: «Записать видео», «Лови файл», «Выбери задачу — и поехали», «Покрути, выбери, скачай», «Тут пока пусто». Сообщения об ошибках — без вины пользователя, с выходом.

## Before → After (что именно меняется vs текущая версия)

| Было | Стало |
|---|---|
| Экспорт после стены шаблонов | Action-док закреплён на превью |
| PNG всегда первой кнопкой | Главное действие = действие задачи |
| Паттерн обещает бесшовность, но не включает | Паттерн ВКЛ. seamless + прячет нетайлы |
| Движение/Искажение — разные имена | Единый язык параметров + мост |
| Только iframe в UI | `<zhizha-bg>` первым, iframe/ссылка рядом |
| Тема UI ≈ палитра сцены визуально | Тема — в углу с подписью, палитра — у фона |
| Mobile: длинный скролл до настроек | Карусель + шит, настройки сразу под превью |
| Нет проверки читаемости под текст | Оверлей читаемости в задаче «Под текст» |
| Нет явного «готово» после скачивания | Состояния exporting / готово / ошибка |
